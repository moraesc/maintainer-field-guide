# 👻 The Phantom Closer

> *"Closing this issue because it's been inactive for 30 days. If this is still relevant, please reopen."*

**Scientific Name:** *Boticus staleclosicus*
**Conservation Status:** Rare (but rising)
**Habitat:** Stale issues, old PRs
**Threat Level to Maintainers:** 🟡 Moderate

---

## Description

The Phantom Closer is a unique species — often not a human at all, but a bot configured to automatically close issues and PRs after a period of inactivity. However, some Phantom Closers *are* human contributors who take it upon themselves to close issues they believe are outdated, resolved, or simply too old.

The ecological impact is debated. Proponents argue that Phantom Closers prevent repository rot by clearing dead issues. Critics point out that closing a valid bug report because nobody responded to it for 60 days doesn't actually fix the bug.

The most controversial subspecies is the **Stale Bot** (*Boticus github-actions*), which has been known to close critical security issues because the maintainer was on vacation.

---

## Behavioral Patterns

- Systematically closes issues older than a configured threshold
- Uses identical template language for every closure
- Does not read the issue content before closing
- Triggers a wave of "this is still relevant!" comments
- Creates more notification noise than it eliminates
- Occasionally closes the issue that the maintainer was actively working on
